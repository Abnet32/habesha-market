Section 01

Project Overview

Habesha Market is an Ethiopian e-commerce project built with plain HTML, CSS, JavaScript, PHP, and MySQL. The documentation below explains how the system is organized, how it stores data, how the pages work, and how the project can be run locally.

🎨
Responsive UI
The front-end adapts to desktop and mobile screens using a compact, card-based layout.
⚡
AJAX Interactions
Search, cart actions, and email validation use asynchronous requests for smoother interactions.
🔧
PHP Backend
Server-side logic is handled by PHP pages and action files without a heavy framework.
🗄️
MySQL Database
The schema includes users, categories, products, carts, orders, order items, and reviews.
🔒
Security Aware
Passwords are hashed and output is escaped to reduce the main risks in the app.
👮
Admin Area
An admin panel supports product, order, and user management.
Section 02

Architecture

The application uses a simple layered structure: HTML/CSS for presentation, JavaScript for client interactivity, PHP for server logic, and MySQL for data storage.

Layer Technology Responsibility
Presentation HTML5 + CSS3 Page structure, layout, and visual styling.
Client Logic Vanilla JavaScript DOM updates, AJAX calls, and form validation.
Server Logic PHP + MySQLi Sessions, requests, business rules, and database access.
Persistence MySQL Stores all project data in relational tables.
Request Flow
  1. The browser requests a PHP page.
  2. The page checks session state and loads required data.
  3. Database queries run through connection.php.
  4. HTML is rendered and escaped before it reaches the browser.
  5. JavaScript binds events for search, cart actions, and validation.
Section 03

Database Schema

The schema is designed for a small e-commerce site but still keeps the data model clean enough for products, orders, carts, and reviews.

Core Tables
Table Purpose
users Accounts for customers and admins.
categories Product group labels.
products Catalog items with price and stock data.
cart_items User cart entries.
orders Order headers and checkout details.
order_items Line items for each order.
reviews Ratings and comments on products.
Relationships
categories → products: one category can contain many products.
users → cart_items: one user can have multiple cart items.
users → orders: one user can place many orders.
orders → order_items: one order can contain many products.
products ↔ users: reviews link both sides together.

1
Import the database

2
Configure the connection
Update connection.php with your local database credentials.
3
Start the server
Use Apache, Nginx, or the PHP built-in server to open the project.
4
Test the demo accounts
Use the sample users in the database to verify login and admin access.
