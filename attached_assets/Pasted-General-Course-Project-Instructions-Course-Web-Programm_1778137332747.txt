General Course Project Instructions
Course: Web Programming
Technologies Required: HTML, CSS, JavaScript, PHP, MySQL
Project Type: Group (max 5 students)
Weight: 20%
Requirement:
You must select your own title and design and develop a fully functional dynamic web
application that solves a real or simulated problem in the Ethiopian context. Your project must
demonstrate:
 Clean, responsive user interface (HTML/CSS – mobile friendly)
 Client-side interactivity (JavaScript – form validation, DOM manipulation, fetch/XHR or
dynamic content)
 Server-side logic (PHP – sessions, request handling, business logic)
 Data persistence (MySQL – at least 3 interconnected tables, CRUD operations)
 Security awareness (basic SQL injection prevention, input sanitization, password hashing
if login exists)
Mandatory Deliverables (to be submitted as a .zip file)
1. Source code (all .html, .css, .js, .php, .sql files)
2. Database export (.sql file with sample data)
3. Project documentation (PDF or .docx) containing:
 System overview and objectives
 Entity Relationship Diagram (ERD) or table relationships
 Screenshots of key pages (UI)
 Instructions on how to set up the project (e.g., "import db.sql, update config.php")
 List of features implemented
 Individual contribution (for group projects)
4. Short demo video (max 3 minutes) OR a live link if hosted (optional but encouraged)
Restrictions
 Do not use heavy frameworks like Laravel, React, or Vue unless permitted (vanilla PHP,
JS, CSS only).
 Do not use mysql_* functions (use mysqli or PDO).
 At least one JavaScript feature must be asynchronous (AJAX/fetch) – e.g., live search,
dynamic dropdown, or real-time validation.
 The project must have a minimum of 5 database tables (including at least one
relationship: one-to-many or many-to-many).
Evaluation Criteria (20 points)
Criteria Points
Title and Functionality (CRUD, sessions, no broken links) 5
UI/UX (responsive, intuitive, local context) 4
Code quality (indentation, naming, comments) 4
JavaScript interactivity (AJAX, validation, dynamic) 2
Documentation & submission completeness 5
Academic Integrity
 Copying full projects from online sources or other groups will result in zero.
 Reusing your own code from previous assignments is allowed but must be extended.
